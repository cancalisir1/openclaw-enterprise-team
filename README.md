# OpenClaw Enterprise Team

This repository contains the architecture implementation for a 12-agent enterprise software development team powered by the OpenClaw gateway. It establishes a workspace-first, evidence-driven, approval-gated, and self-improving operational structure.

## Structure

- `platform/`: Control plane files (`openclaw.json`), security profiles, and global hooks.
- `workspaces/`: Individual operational directories for the 12 core agents:
  - `chief-orchestrator`, `product-owner`, `system-architect`, `backend-lead`
  - `frontend-lead`, `qa-evidence`, `platform-sre`, `security-governor`
  - `research-librarian`, `memory-steward`, `eval-redteam`, `proactive-ops`
- `shared/`: Shared templates, global evals (Promptfoo), runbooks, and OpenProse workflows.
- `registry/`: Security-governed skills storage (Approved/Candidate/Rejected).
- `labs/`: Optimization tests (autoresearch loop) and Simulation environments (MiroFish-inspired).
- `docs/`: Superpowers-inspired product delivery lifecycle (PRD -> ADR -> PLAN -> EVIDENCE).

## Operating Principles
1. **Workspace-First**: Agent markdown state (`AGENTS.md`, `SOUL.md`, `MEMORY.md`, `.learnings`) is the ultimate source of truth. Remote logs (e.g. OpenViking) only supplement this.
2. **Delivery Contract**: No code changes are merged without an EVID (Evidence) artifact.
3. **Approval Gated**: Direct host execution commands and sensitive tool usages are strictly bound by the `exec-approvals` structure.
4. **Learn & Promote**: Memory Steward continuously analyzes `.learnings` generated by team members and upgrades the team’s rules dynamically.

## Quickstart

1. Spin up the OpenClaw server pointing to `platform/openclaw.json`.
2. Connect OpenViking memory plugin if required.
3. Initialize a session with the `chief-orchestrator` and present a PRD under `docs/prd/`.
